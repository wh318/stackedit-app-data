# 初识 Linux

## 一、 Linux 核心思维

Linux 与 Windows 最大区别在于其高度的逻辑性和透明度。

-   **根文件系统**：没有分区的概念（如 C/D 盘），所有硬件、驱动和数据均挂载在根目录 `/` 下。
    
-   **一切皆文件**：无论是硬件设备（磁盘、键盘）还是进程，在 Linux 中都以文件形式存在，这意味着可以用同一套工具进行管理。
    
-   **命令行优先级**：虽然有图形界面，但高效的自动化管理和服务器运维几乎完全依赖终端（Terminal）。
    

## 二、 基础操作指令

掌握以下核心指令即可完成 80% 的日常任务：

### 1. 文件与目录管理

-   `ls`：列出当前目录内容（`-a` 查看隐藏文件，`-l` 查看详细属性）。
    
-   `cd`：切换路径（`cd ..` 返回上级，`cd ~` 返回用户主目录）。
    
-   `pwd`：显示当前工作目录的完整路径。
    
-   `mkdir`：创建新文件夹。
    
-   `cp`：复制文件或目录（复制目录需加 `-r`）。
    
-   `mv`：移动文件或重命名。
    
-   `rm`：删除文件（`rm -rf` 强制删除目录，操作需极其谨慎）。
    

### 2. 系统权限与用户

-   `sudo`：以系统管理员（root）权限执行命令。
    
-   `chmod`：修改文件权限（如 `chmod 755 file`）。
    
-   `chown`：修改文件所属用户或组。
    

### 3. 查看文件内容

-   `cat`：一次性读取并打印整个文件。
    
-   `less`：分页查看大文件，支持上下翻页。
    
-   `head` / `tail`：查看文件的前几行或后几行（`tail -f` 常用于实时监控日志）。

## 三、 文件权限体系

执行 `ls -l` 后，文件属性的第一列（如 `-rwxr-xr--`）决定了访问控制：

| 符号 | 权限 | 说明 | 数值 |
| :-: | :-: | 
| **r** | Read | 读取内容 / 查看目录列表 | 4 |
| **w** | Write | 修改内容 / 在目录内增删文件 | 2 |
| **x** | Execute | 执行文件 / 进入目录 | 1 |

匯出到試算表

-   **权限组合**：前三位为属主（Owner），中间三位为属组（Group），最后三位为其他人（Others）。
    
-   **例子**：755 代表属主拥有全部权限 (4+2+1=7)，而组员和其他人只有读和执行权限 (4+1=5)。

## 四、 必备工具与软件管理

### 1. 文本编辑器：Vim

在 Linux 中修改配置文件必不可少。

-   **模式切换**：默认是命令模式，按 `i` 进入编辑模式；按 `Esc` 退回命令模式。
    
-   **保存退出**：在命令模式下输入 `:wq` 存盘退出，`:q!` 强制不保存退出。
    

### 2. 包管理器

根据发行版的不同，安装软件的工具也不同：

-   **Ubuntu/Debian**：`sudo apt update`（更新列表），`sudo apt install <包名>`。
    
-   **RHEL/CentOS**：`sudo yum install <包名>` 或 `dnf`。
    

----------

## 五、 操作效率技巧

-   **Tab 键自动补全**：输入命令或路径的前几个字母后按 Tab，可极大减少输入错误。
    
-   **管道符 `|`**：将前一个命令的输出作为后一个命令的输入。例如：`ls | grep ".txt"`（在文件列表中搜索所有 .txt 文件）。
    
-   **通配符 `*`**：代表任意字符。例如：`rm *.log` 删除所有日志文件。
    
-   **历史记录**：使用 `history` 查看执行过的指令，或按键盘「上/下」键快速切换。
    

----------

## 六、 学习路径建议

1.  **环境搭建**：初学者建议在虚拟机（VMware/VirtualBox）中安装 Ubuntu 或 Mint。
    
2.  **放弃鼠标**：尝试在终端中完成新建、删除、编辑和移动文件的全过程。
    
3.  **理解流程**：重点掌握进程管理（`ps`, `top`）和网络基础（`ifconfig`, `ping`）。
